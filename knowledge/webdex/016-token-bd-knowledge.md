## 2. Filosofia de Escassez

O protocolo **rejeita burning artificial**. O mecanismo é "escassez natural" (natural scarcity):

- Staking obrigatório retira tokens de circulação de forma permanente enquanto o trader opera
- Sem queima = sem deflação artificial = valor sustentável pelo uso real
- O supply nunca cresce além de 369.369.369 BD — hard cap imutável
- Nenhum novo token pode ser criado após o supply inicial

> "O valor não é diluído por criação arbitrária de novos tokens"

## 4. Casos de Uso do Token BD

### 4.1 Fee de Protocolo (principal driver de demanda)
- **0.00963 BD** por operação de arbitragem triangular
- **9.63%** do valor de vendas de cursos (sem afiliado presente)
- **19.26%** do valor de vendas de cursos (com afiliado)
- **9.63%** das tarefas executadas no Task Marketplace

### 4.2 Token Pass (Acesso via Staking)
- Usuários devem fazer stake de exatamente **96.3 BD** para desbloquear:
  - Automação de arbitragem
  - Publicação de cursos
  - Participação no programa de afiliados
  - Execução de tarefas no marketplace
- O stake cria "escassez natural" — tokens travados, não queimados

### 4.3 Governança via V.A.L.T.
- **V.A.L.T.** = Vínculo Ativo de Longo Termo
- Poder de voto proporcional ao tamanho do stake e duração do compromisso
- Permite participação em decisões do protocolo
- Apenas a alocação para desenvolvedores (9%) é imutável — todo o resto pode ser votado

### 4.4 Meio de Troca Interno
- Única moeda aceita no marketplace interno de cursos e tarefas
- Garante demanda constante e orgânica pelo token
