## 🎯 Goal

Corrigir 3 bugs críticos no worker `agendador_21h` (race condition / deadlock, timezone UTC vs BRT, double-message) e expandir o alcance das notificações para todos os 18 usuários ativos em todos os canais disponíveis (2 Discord + 18 Telegram).

---

## 📋 Context

O `agendador_21h` em `webdex_workers.py` (container `ocme-monitor`) apresentava:

1. **Race condition / deadlock** — flag `"pending"` sem timestamp permitia deadlock permanente em restart do container
2. **Timezone errado** — `protocol_ops.ts` armazena UTC mas a query usava `_ciclo_21h_since()` que retorna BRT (UTC-3), causando sub-contagem de ~3h de operações
3. **Double-message / exclusão indevida** — lógica de skip de usuários que já receberam relatório pessoal impedia broadcast do relatório protocolo para 2/3 dos usuários
4. **Canal único** — relatório protocolo ia apenas para #relatório-diário (Discord); sem cobertura de #webdex-on-chain nem Telegram

Smith identification: COMPROMISED → patches aplicados → CONTAINED

## ✅ Acceptance Criteria

- [x] Flag `pending:TIMESTAMP` com TTL de 10 minutos — deadlock resolvido
- [x] `_dt_lim = BRT + timedelta(hours=3)` — query em UTC correto

## 🔧 Mudanças Aplicadas

### `packages/monitor-engine/webdex_workers.py`

**State machine flag (3 estados):**
- `""` — disponível para disparo
- `"pending:TIMESTAMP"` — em andamento, TTL 10 min
- `"ok"` — enviado e confirmado

**Query corrigida (UTC):**
```python
_dt_lim_brt = _ciclo_21h_since()  # retorna BRT
_dt_lim = (datetime.strptime(_dt_lim_brt, ...) + timedelta(hours=3)).strftime(...)
```

**Query simplificada (sem gás):**
- `COUNT(DISTINCT wallet)`, `COUNT(CASE WHEN profit>0)`, `COUNT(*)`, `SUM(fee_bd)`, `SUM(profit WHERE >0)`
- TVL via `fl_snapshots` — `SUM(lp_usdt_supply + lp_loop_supply)` do snapshot mais recente

**Broadcast Telegram — todos ativos:**
```python
for (uid,) in _tg_users:
    send_html(uid, _tg_msg)   # sem skip por last_rep
```

**Exception handler condicional:**
```python
if get_config(_discord_21h_key, "") != "ok":
    set_config(_discord_21h_key, "")  # só reseta se Discord não confirmado
```

### `packages/monitor-engine/webdex_discord_sync.py`

**`notify_protocolo_relatorio`** — nova assinatura simplificada:
- Removidos: `pol_price`, `p_trades`, `p_gas_pol`, `p_gas_usd`, `p_ganhos`, `p_perdas`, `bd_alltime`, `proto_count`
- Adicionados: `tvl_usd`, `bd_periodo`, `p_wr`, `p_bruto`
- Método: `_async_post` → `_post_webhook` (síncrono, 3 tentativas)
- Top 5: row indexing com bounds guard `len(row) > 2`

**`notify_protocolo_relatorio_telegram`** *(nova)*:
- Retorna string HTML para broadcast Telegram
- Mesma estrutura da versão Discord
- Top 5 com bounds guard

**`notify_protocolo_relatorio_onchain`** *(nova)*:
- Resumo compacto → `#webdex-on-chain`
- Async (fire-and-forget, canal secundário)
- Campos: TVL, BD, traders, WR, P&L bruto
