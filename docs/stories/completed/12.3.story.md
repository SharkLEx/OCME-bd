# Story 12.3 — Streaming Responses Discord

**Status:** ✅ Done
**Epic:** 12 — bdZinho Intelligence v3
**Prioridade:** SHOULD
**Criado:** 2026-03-19

---

## 🎯 Goal

Implementar streaming progressivo de respostas no Discord bot (bdZinho).
Em vez de aparecer a resposta completa de uma vez após latência total,
o usuário vê a resposta sendo construída em tempo real (edição progressiva da mensagem).

---

## 📋 Context

- Canal atual: `on_message` → `ai_handler.get_ai_response()` (sync, bloqueia até resposta completa)
- Latência típica: 3–8s de espera com tela em branco
- UX alvo: placeholder "💭 *Processando...*" aparece imediatamente, texto vai sendo editado à medida que chunks chegam
- Discord rate limit: ~5 edits/5s por mensagem — throttle em ≥ 800ms

---

## 📦 Acceptance Criteria

### AC1 — AsyncOpenAI + stream_ai_response()
- [x] `AsyncOpenAI` lazy singleton em `ai_handler.py`
- [x] `stream_ai_response()` — async generator que yields chunks de texto
- [x] Memória gravada APÓS streaming completo (user antes, assistant depois)
- [x] `stream_options={"include_usage": True}` para capturar tokens no chunk final
- [x] Fallback: se stream lançar exceção, relança para bot.py tratar

### AC2 — Progressive Edit em bot.py
- [x] Placeholder `"💭 *Processando...*"` enviado imediatamente (sem await de IA)
- [x] Edição a cada ≥ 800ms com cursor `▌` durante streaming
- [x] Edit final sem cursor, com resposta completa (truncada a 1900 chars se necessário)
- [x] Auto-escalação (confidence < 0.4) aplicada no texto final
- [x] `db_handler.save_ai_response()` chamado após streaming completo

### AC3 — Graceful Degradation
- [x] Se `stream_ai_response` lançar exceção → edita placeholder com mensagem de erro
- [x] `HTTPException` em edits intermediários → silenciado (skip edit, continua stream)
- [x] Logging de latência total e tokens (quando disponíveis)
