# Story 12.4 — Persona Engine

**Status:** ✅ Done
**Epic:** 12 — bdZinho Intelligence v3
**Prioridade:** SHOULD
**Criado:** 2026-03-19

---

## 🎯 Goal

Tornar o bdZinho contextualmente inteligente: detecta automaticamente o modo da
conversa (win/loss/onboarding/technical/default) e adapta o tom da resposta
injetando hints direcionais no prompt — sem chamadas extras à IA.

---

## 📋 Context

- `voice_discord.py` já tem SYSTEM_PROMPT com descrição dos modos de personalidade
- `build_prompt()` recebe `user_message`, `protocol_ctx`, `user_env` mas não adapta o tom
- A detecção deve ser leve: keyword matching + heurísticas, sem custo extra de tokens
- `history_len` (# mensagens anteriores) indica se usuário é novo (< 3 msgs)

---

## 📦 Acceptance Criteria

### AC1 — detect_persona_mode()
- [x] Detecta: 'win', 'loss', 'onboarding', 'technical', 'default'
- [x] Sinais de win: P&L positivo no ctx, keywords "lucro/ganho/positivo/cresceu"
- [x] Sinais de loss: ciclo negativo, keywords "anomalia/perda/negativo"
- [x] Sinais de onboarding: history_len < 3 + keywords de início
- [x] Sinais de technical: keywords "contrato/endereço/0x/hash/gas"
- [x] Sem chamada extra à IA — apenas string matching

### AC2 — get_persona_hint() + build_prompt() atualizado
- [x] _PERSONA_HINTS dict com hint específico por modo
- [x] build_prompt() aceita `history_len=0` e injeta hint no prompt
- [x] Modo 'default' → hint vazio (sem overhead)

### AC3 — ai_handler.py passa history_len
- [x] get_ai_response() e stream_ai_response() passam len(history) para build_prompt()
