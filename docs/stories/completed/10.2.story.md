## 🎯 Goal

Commitar e deployar as atualizações de `webdex_discord_sync.py`, `notification_engine.py` e `webdex_anomaly.py` — que completam a **migração de cores hardcoded** para `design_tokens.py` no monitor-engine, tornando toda a camada de notificações Discord on-brand e consistente com o Design System WEbdEX.

---

## 📋 Context

Após o Epic 9, o `packages/orchestrator/discord/design_tokens.py` define os tokens oficiais da marca. O monitor-engine ainda possuía cores hardcoded em vários lugares:

- `webdex_discord_sync.py`: cores inline nos embeds de operações/swaps/token
- `notification_engine.py`: melhorias e uso de tokens para notificações proativas
- `webdex_anomaly.py`: pequeno fix de 10 linhas relacionado a cores/alertas

O @dev já implementou todas as mudanças (+54L +94L +10L). Esta story formaliza o commit.

### Migração de cores — padrão

```python
# ANTES (hardcoded)
color = 0x00FF88  # verde P&L positivo
color = 0xFF4444  # vermelho P&L negativo

# DEPOIS (design tokens)
from design_tokens import SUCCESS, ERROR
color = SUCCESS   # 0x00FFB2 — on-brand
color = ERROR     # 0xFF4455 — on-brand
```

### Tokens Discord disponíveis (design_tokens.py)

```python
PINK_LIGHT  = 0xFB0491   # accent primário
RED_LIGHT   = 0xD90048   # accent secundário
SUCCESS     = 0x00FFB2   # P&L positivo
WARNING     = 0xFF8800   # atenção
ERROR       = 0xFF4455   # crítico
PRO_PURPLE  = 0xE040FB   # tier PRO
CHART_BLUE  = 0x00D4FF   # gráficos
```
