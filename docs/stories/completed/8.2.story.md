# Story 8.2 — FastAPI Gateway + Webhook Router

**Epic:** Epic 8 | **Status:** ✅ Done | **Data:** 2026-03-15

## 🎯 Goal
FastAPI como camada central do orquestrador:
- Recebe webhooks de plataformas externas (Instagram, WhatsApp futuramente)
- Expõe API interna para o Discord bot e n8n
- Rota eventos para handlers corretos
- Health check e status endpoint

## Stack
- FastAPI + Uvicorn (Python 3.11)
- Roda no container `orchestrator-api` (nova service no docker-compose)
- Porta interna 8000, exposta via Traefik em `api.webdex.app` (futuro)

## ✅ Acceptance Criteria
- [ ] `GET /health` retorna `{"status": "ok", "version": "1.0.0"}`
- [ ] `GET /status` retorna dados reais do protocolo (TVL, ops, capital) — lê OCME DB read-only
- [ ] `POST /webhook/discord` recebe eventos Discord → loga em PostgreSQL
- [ ] `POST /webhook/instagram` recebe webhook Meta com verificação (futuro: stub que retorna challenge)
- [ ] `GET /metrics` retorna contadores básicos (eventos recebidos por plataforma)
- [ ] Dockerfile + integrado no docker-compose.yml do orchestrator
- [ ] Healthcheck no docker-compose

## Dev Notes
- OCME DB montado read-only em `/app/data/webdex_v5_final.db` (igual ao discord)
