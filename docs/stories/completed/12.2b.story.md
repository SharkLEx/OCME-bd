# Story 12.2b — Tool Use Discord (port do Telegram)

**Status:** ✅ Done
**Epic:** 12 — bdZinho Intelligence v3
**Prioridade:** MUST
**Criado:** 2026-03-19

---

## 🎯 Goal

Portar o Function Calling do Telegram (Story 12.2) para o Discord bot.
bdZinho passa a buscar dados on-chain em tempo real durante a conversa,
sem chamadas extras de API — usa SQLite OCME (já montado read-only no container).

---

## 📋 Context

- `webdex_tools.py` (monitor-engine): tool use completo para Telegram, usa `webdex_db` (SQLite do container monitor)
- Discord container: tem acesso ao mesmo SQLite via volume `/ocme_data:ro` → `protocol_context._conn()`
- `db_handler.get_user_wallet(user_id)` → wallet do usuário Discord (PostgreSQL)
- `stream_ai_response()` já streaming com AsyncOpenAI — precisa suportar tool_calls loop

## 📦 Acceptance Criteria

### AC1 — webdex_tools_discord.py
- [x] `DISCORD_TOOLS`: 3 tools no formato OpenAI Function Calling
- [x] `get_protocol_metrics(metric)` → SQLite fl_snapshots + protocol_ops
- [x] `get_user_portfolio()` → wallet via PostgreSQL + capital/snapshots via SQLite (sem wallet conectada → mensagem para /conectar-wallet)
- [x] `get_market_context()` → gas via web3 Polygon RPC + POL price via SQLite
- [x] Circuit breaker (3 falhas → OPEN 5min), rate limit (20 calls/h), timeout por tool
- [x] `execute_tool_discord(name, args, user_id)` → dispatcher com circuit breaker

### AC2 — ai_handler.py: tool_calls no stream
- [x] `stream_ai_response()` passa `tools=DISCORD_TOOLS` para a API
- [x] Acumula `delta.tool_calls` durante streaming (não yielda para Discord)
- [x] Quando `finish_reason == "tool_calls"` → executa tools via `run_in_executor` (não bloqueia event loop)
- [x] Monta mensagem assistant com tool_calls + mensagens tool → segundo streaming call
- [x] Max 3 iterações de tool_calls
- [x] Graceful degradation: se `webdex_tools_discord` não importar → funciona sem tools

### AC3 — UX sem alteração no bot.py
- [x] Nenhuma mudança em bot.py necessária — `stream_ai_response` mantém mesma interface
