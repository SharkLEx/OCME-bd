# Story 8.6 — Community Hub: Atendimento Centralizado

**Epic:** Epic 8 | **Status:** ✅ Done | **Data:** 2026-03-15

## 🎯 Goal
Sistema de atendimento centralizado no Discord:
- Comando `/suporte` — cria ticket, loga no PostgreSQL, notifica canal admin
- FAQ instantâneo — respostas rápidas para perguntas comuns sem chamar IA
- Auto-escalação — quando bot não tem confiança na resposta, avisa admin
- Histórico de tickets acessível via FastAPI (`GET /support/tickets`)

## ✅ Acceptance Criteria
- [ ] `/suporte [mensagem]` cria ticket no PostgreSQL e posta no canal admin
- [ ] `/faq` lista as 5 perguntas mais frequentes do protocolo
- [ ] AI responder inclui score de confiança — se < 0.5 → escala
- [ ] `GET /support/tickets` retorna lista de tickets (last 50)
- [ ] `GET /support/tickets/{id}` retorna ticket específico
- [ ] Tabela `support_tickets` criada no PostgreSQL

## Dev Notes
- Canal admin: env var `DISCORD_ADMIN_CHANNEL_ID`
- Score de confiança: heurística baseada em keywords e tamanho da resposta
- FAQs hardcoded no bot (não requer DB) — editável em `faq_data.py`
- Tickets fechados manualmente pelo admin com reação ✅
