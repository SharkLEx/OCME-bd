# Story 8.9 — WhatsApp Canal Integration (Business API)

## Story
Como operador do WEbdEX Orchestrator, quero integrar o WhatsApp Business Cloud API ao sistema para receber mensagens, responder com IA contextual e persistir eventos — operando em modo stub quando não configurado e ativando automaticamente quando as credenciais forem fornecidas.

## Acceptance Criteria
- [x] GET /webhook/whatsapp — verifica challenge Meta (mesmo padrão Instagram)
- [x] POST /webhook/whatsapp — recebe mensagens, status updates, persiste em whatsapp_events
- [x] whatsapp/client.py — send_message(), send_template(), mark_read()
- [x] whatsapp/handlers.py — processa text, image, audio, document, interactive
- [x] GET /whatsapp/status — verifica configuração e modo (live/stub)
- [x] GET /whatsapp/events — lista últimos eventos recebidos
- [x] Migration 006_whatsapp_events.sql aplicada no PostgreSQL
- [x] Dockerfile e docker-compose incluem módulo whatsapp/
- [x] Smoke test: GET /whatsapp/status retorna mode: stub quando sem token
