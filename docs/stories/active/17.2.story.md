## 🎯 Goal

Publicar automaticamente no TikTok os vídeos MP4 1080x1920 gerados pelo Creatomate para o ciclo 21h.

- Mesmo vídeo já gerado para Instagram/Telegram → publicar no TikTok sem re-render
- Cadência: 1 post/ciclo 21h (automático, sem aprovação manual)
- Fallback: se vídeo não disponível → postar text-only com dados do ciclo

---

## 📋 Context

**API:** TikTok Content Posting API v2
**Auth:** OAuth 2.0 com `video.upload` + `video.publish` scopes
**Rate limit:** 100 posts/dia (Business account)
**Formato de vídeo:** MP4, max 10min, max 500MB — Creatomate gera 1080x1920 (vertical ✅)
**Credenciais necessárias (Alex deve criar em developers.tiktok.com):**
- `TIKTOK_CLIENT_KEY`
- `TIKTOK_CLIENT_SECRET`
- `TIKTOK_ACCESS_TOKEN`

**Fluxo de upload TikTok Content Posting API:**
1. `POST /v2/post/publish/video/init/` → obtém `upload_url` + `publish_id`
2. `PUT {upload_url}` → faz upload do arquivo MP4
3. `GET /v2/post/publish/status/fetch/` → polling até `PUBLISH_COMPLETE`

## 🛠️ Dev Notes

**Padrão de cliente:**
```python
class TikTokClient:
    def __init__(self):
        self.access_token = os.environ.get('TIKTOK_ACCESS_TOKEN', '')
        self.client_key = os.environ.get('TIKTOK_CLIENT_KEY', '')
        self._configured = bool(self.access_token and self.client_key)

    def is_configured(self) -> bool:
        return self._configured

    def upload_video(self, video_path: str, caption: str) -> dict:
        if not self._configured:
            logger.info('[tiktok] stub mode — vídeo não publicado')
            return {'stub': True}
        # Step 1: init upload
        init_resp = requests.post(
            'https://open.tiktokapis.com/v2/post/publish/video/init/',
            headers={'Authorization': f'Bearer {self.access_token}'},
            json={
                'post_info': {'title': caption, 'privacy_level': 'PUBLIC_TO_EVERYONE'},
                'source_info': {'source': 'FILE_UPLOAD', 'video_size': os.path.getsize(video_path),
                                'chunk_size': os.path.getsize(video_path), 'total_chunk_count': 1}
            }
        )
        upload_url = init_resp.json()['data']['upload_url']
        publish_id = init_resp.json()['data']['publish_id']
        # Step 2: upload file
        with open(video_path, 'rb') as f:
            requests.put(upload_url, data=f,
                headers={'Content-Type': 'video/mp4',
                         'Content-Range': f'bytes 0-{os.path.getsize(video_path)-1}/{os.path.getsize(video_path)}'})
        return {'publish_id': publish_id}
```

**Nota sobre OAuth:** TikTok Content Posting API exige conta Business + aprovação do app. O access_token tem validade de 24h (renovação via refresh_token). Para MVP, usar access_token manual (renovar mensalmente).

**Vídeo Creatomate:** Gerado em `creatomate_worker.py` e salvo temporariamente. O path é passado via `notify_tiktok(video_path=...)`. Se o worker não gerou vídeo (erro Creatomate), TikTok post é skipado silenciosamente.
