## 🎯 Goal

Publicar automaticamente no Twitter/X os eventos do WEbdEX Protocol OS:
- Resultado do ciclo 21h (PnL, TVL, operações)
- Milestones on-chain (TVL record, N ops, novo subscriber)
- Alertas críticos de mercado (anomalias severas)

Sem aprovação manual — 100% automatizado, mesma cadência do Discord/Telegram.

---

## 📋 Context

**API:** Twitter API v2 — `POST /2/tweets`
**Auth:** OAuth 2.0 Bearer Token (app-level) para post
**Rate limit:** 17 tweets / 24h (free tier) | 1500 / mês (basic tier - $100/mês)
**Formato:** Plain text + emojis + hashtags (sem Markdown)

**Credenciais necessárias (Alex deve criar em developer.twitter.com):**
- `TWITTER_BEARER_TOKEN`
- `TWITTER_API_KEY` + `TWITTER_API_SECRET`
- `TWITTER_ACCESS_TOKEN` + `TWITTER_ACCESS_TOKEN_SECRET`

## 📦 Acceptance Criteria

### AC1 — Cliente Twitter API v2
- [ ] Criar `packages/orchestrator/twitter/client.py`:
  - `post_tweet(text: str) -> dict` — publica tweet, retorna tweet_id
  - Rate limit tracking: contador diário em config table
  - Stub mode: quando sem credenciais, loga e retorna sem erro
- [ ] Sem dependências externas além de `requests` (já presente)
- [ ] `TWITTER_BEARER_TOKEN` via env var

### AC2 — Formatador de tweets
- [ ] Criar `packages/orchestrator/twitter/formatter.py`:
  - `format_ciclo_tweet(data: dict) -> str` — max 280 chars
    ```
    📊 WEbdEX Ciclo 21h
    TVL: $1.5M (+2.3%)
    Operações: 3.756
    PnL Top: +12.4%

    #WEbdEX #DeFi #Polygon
    ```
  - `format_milestone_tweet(milestone: str, value: str) -> str`
  - `format_alert_tweet(alert_type: str, details: str) -> str`
- [ ] Truncagem automática se > 280 chars (sem cortar no meio de palavra)
- [ ] Hashtags fixas: `#WEbdEX #DeFi #Polygon`

### AC3 — Integração com notification_engine.py
- [ ] Adicionar `notify_twitter()` no `notification_engine.py`:
  - Chamado junto com Discord/Telegram nos eventos
  - Graceful degradation: falha no Twitter não bloqueia outras notificações
- [ ] Integrar no ciclo 21h: `metrics_worker.py` chama `notify_twitter()` após embed Discord

### AC4 — Deduplicação
- [ ] Não postar o mesmo ciclo 2x (verificar hash do conteúdo em config table)
- [ ] Cooldown de 1h entre tweets do mesmo tipo (evitar spam em falhas de retry)

### AC5 — Deploy
- [ ] Adicionar vars ao `.env.example`:
  ```
  TWITTER_BEARER_TOKEN=
  TWITTER_API_KEY=
  TWITTER_API_SECRET=
  TWITTER_ACCESS_TOKEN=
  TWITTER_ACCESS_TOKEN_SECRET=
  ```
- [ ] Smoke test: `python -c "from twitter.client import TwitterClient; c = TwitterClient(); print(c.is_configured())"` → True/False sem erro

**Padrão de cliente:**

```python
        self._configured = bool(self.bearer)

    def is_configured(self) -> bool:
        return self._configured

    def post_tweet(self, text: str) -> dict:
        if not self._configured:

```

**Nota:** Twitter API v2 `POST /2/tweets` requer OAuth 1.0a User Context (não Bearer Token) para escrever. Usar `requests-oauthlib` ou implementar OAuth 1.0a manualmente com `hmac` (stdlib).
