## 🎯 Goal

Commitar e deployar o `metrics_worker.py` — worker assíncrono que faz snapshots periódicos do protocolo no PostgreSQL e envia notificação proativa diária no canal `#bdzinho-ia` do Discord após o ciclo 21h BRT.

---

## 📋 Context

O Epic 9 entregou o `notification_engine.py` com a infraestrutura base de notificações. Esta story adiciona:

1. **`metrics_worker_loop()`** — snapshot do protocolo no PostgreSQL a cada 10 minutos, usando `get_status_embed_data()` + `db_handler`
2. **`ciclo_21h_worker(bot)`** — notificação diária no `#bdzinho-ia` entre 00:10–00:59 UTC (21:10–21:59 BRT), com deduplicação via PostgreSQL + in-memory set
3. **Embed de resumo do ciclo** — usa `embed_builder.py` do Discord com dados reais do PostgreSQL (P&L delta, ops, TVL, wallets), com gráfico TVL anexado via `generate_chart()`

### Tabela de banco necessária

```sql
CREATE TABLE IF NOT EXISTS protocol_snapshots (
    id          BIGSERIAL PRIMARY KEY,
    ts          TIMESTAMPTZ DEFAULT NOW(),
    tvl_total   NUMERIC(20,4),
    tvl_ag      NUMERIC(20,4),
    tvl_bd      NUMERIC(20,4),
    ops_count   BIGINT,
    ops_profit  NUMERIC(20,8),
    cap_wallets INTEGER,
    cap_total   NUMERIC(20,4),
    positions   INTEGER
);
```

A tabela `platform_events` (já existente desde Story 8.1) é usada para persistir a flag `ciclo_21h_notificado`, prevenindo duplicatas após restart do container na janela de notificação.
