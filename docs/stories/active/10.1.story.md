## 🎯 Goal

Commitar e deployar o `telegram_design_tokens.py` — sistema completo de Design Tokens para mensagens Telegram — e a versão atualizada de `webdex_workers.py` que usa esses tokens para formatar o relatório de ciclo 21h com identidade visual WEbdEX consistente.

---

## 📋 Context

O relatório 21h do bot OCME_bd (Telegram) usa formatação manual e hardcoded. O @dev já criou `telegram_design_tokens.py` seguindo o mesmo padrão do `design_tokens.py` do Discord (Epic 9):

- Namespace `EMOJI` — emojis organizados por categoria semântica
- Namespace `SEP` — separadores (━━━, ─────, ├─, └─)
- Namespace `HDR` — headers pré-montados (ciclo_21h, mybdBook, etc.)
- Namespace `MSG` — textos recorrentes fixos
- Formatadores: `format_currency()`, `format_pct()`, `format_pol()`, `format_bd()`, etc.
- Barras visuais: `progress_bar()`, `winrate_bar()`, `ops_bar()`
- Blocos pré-montados: `bloco_pnl_traders()`, `bloco_gas()`, `bloco_receita()`, `bloco_top_traders()`, etc.

O `webdex_workers.py` foi atualizado (+88 linhas) para usar esses tokens no lugar de strings hardcoded.

# Namespaces
EMOJI = _EmojiNS()     # status, métricas, blockchain, conquistas, alertas

HDR   = _HdrNS()       # ciclo_21h(), mybdbook(), protocolo_ao_vivo(), swapbook(), token_bd(), secao()

# Constantes

format_webdex(value) → str
format_int(value) → str
